## defect 1: deleted error import

### what
test file `createQueueWithResilientRemoteConsumer.test.ts` imported `UnexpectedCodePathError` from a relative path to a deleted file.

### how (diff against origin/main)
declapract practice `errors.bad-practice.own-implementation` deleted `src/utils/errors/UnexpectedCodePathError.ts` and added `helpful-errors` package.

### why (root cause)
declapract auto-fix deleted the custom error file but did not update imports that referenced it via local path. the practice only handles package imports (e.g., `from '@ehmpathy/error-fns'`), not relative imports to deleted files.

### fix
```diff
- import { UnexpectedCodePathError } from '../../utils/errors/UnexpectedCodePathError';
+ import { UnexpectedCodePathError } from 'helpful-errors';
```

## defect 2: cache api incompatibility

### what
type error: `SimpleInMemoryCache` not assignable to cache option type because cache `set` method options changed.

### how (diff against origin/main)
- `simple-in-memory-cache` upgraded from 0.3.0 to 0.4.3
- api changed from `secondsUntilExpiration` to `expiration` (iso-time format)
- `with-simple-caching` 0.11.1 (old package name) expected old cache api

### why (root cause)
declapract upgraded `simple-in-memory-cache` but not the cache wrapper package. additionally, the cache wrapper package was renamed from `with-simple-caching` to `with-simple-cache`.

### fix
1. replaced `with-simple-caching` with `with-simple-cache@0.16.2` (package was renamed)
2. updated import from `withSimpleCaching` to `withSimpleCache`
3. updated cache creation call:
```diff
- cache: createCache({
-   defaultSecondsUntilExpiration: 0,
- }),
+ cache: createCache({
+   expiration: { seconds: 0 },
+ }),
```

## defect 3: orphaned devDependencies

### how (diff against origin/main)

### why (root cause)

- `@tsconfig/node-lts-strictest` — obsolete, tsconfig extends different packages
- `core-js` — obsolete, polyfills not needed
- `ts-jest` — replaced by `@swc/jest`
- `ts-node` — replaced by `tsx` and `esbuild-register`

### fix
removed orphaned packages from package.json devDependencies.
