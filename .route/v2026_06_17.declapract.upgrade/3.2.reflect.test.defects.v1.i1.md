## defect 1: deleted error import

### classification: practice bug

### analysis

declapract practice `errors` deleted the custom error file `src/utils/errors/UnexpectedCodePathError.ts` but did not update imports that referenced that local path.

the test file imported from a relative path:
```ts
import { UnexpectedCodePathError } from '../../utils/errors/UnexpectedCodePathError';
```

the practice only updates imports that match the package import pattern (e.g., `from '@ehmpathy/error-fns'`), not local file imports to deleted files.

### citation

```
gh search code --repo ehmpathy/declapract-typescript-ehmpathy "helpful-errors"
```

```
ehmpathy/declapract-typescript-ehmpathy:src/practices/errors/bad-practices/error-fns/src/**/*.ts.declapract.ts: "from 'helpful-errors'",
```

the practice handles package imports but not local file imports to custom error implementations.

### recommendation

the error migration practice should also update local imports to deleted error files:
- find: imports from `*/errors/*Error` (local paths)
- replace: import from `helpful-errors`

## defect 2: cache api incompatibility

### classification: practice bug

### analysis

declapract practice `cache/best-practice` enforces minimum version for `simple-in-memory-cache` (0.4.3) but does not enforce migration from `with-simple-caching` (deprecated) to `with-simple-cache` (renamed package).

the cache api changed between 0.3.x and 0.4.x:
- old: `{ defaultSecondsUntilExpiration: number }`
- new: `{ expiration: { seconds: number } }`

`with-simple-caching` was renamed to `with-simple-cache`. the new package at 0.16.2 supports the new api.

### citation

```
gh search code --repo ehmpathy/declapract-typescript-ehmpathy "simple-in-memory-cache"
```

```
ehmpathy/declapract-typescript-ehmpathy:src/practices/cache/best-practice/package.json: "simple-in-memory-cache": "@declapract{check.minVersion('0.4.3').ifInstalled()}",
```

no equivalent check exists for the cache wrapper package migration.

### recommendation

add a bad-practice to migrate from `with-simple-caching` to `with-simple-cache`:

```json
"with-simple-cache": "@declapract{check.minVersion('0.16.0').ifInstalled()}"
```

## defect 3: orphaned devDependencies

### classification: repo quirk

### analysis

declapract migrated from eslint/prettier/ts-jest to biome/@swc/jest but left orphaned packages in package.json:
- `@trivago/prettier-plugin-sort-imports`
- `@tsconfig/node-lts-strictest`
- `core-js`
- `ts-jest`
- `ts-node`

### citation

```
gh search code --repo ehmpathy/declapract-typescript-ehmpathy "@swc/jest"
```

```
ehmpathy/declapract-typescript-ehmpathy:src/practices/lint/bad-practices/depcheck-denylisted-ignores/.declapract.readme.md: - ts-jest: use @swc/jest instead
```

the practice documents the migration path but relies on depcheck to flag orphaned packages rather than auto-remove them.

### rationale for repo quirk classification

this is a repo quirk because:
1. depcheck correctly flagged the orphaned packages
2. the practice documents the migration path
3. auto-removal would be aggressive for declapract (some repos may still need ts-jest)

the fix was straightforward: remove the flagged packages.
