## hazard 2: cache api changed

### what changed
`simple-in-memory-cache` upgraded from 0.3.0 to 0.4.3.
the cache options api changed from `secondsUntilExpiration` to `expiration` with iso-time format.

### why it breaks
- `createQueueWithResilientRemoteConsumer.ts` uses old api:
  ```ts
  cache: new SimpleInMemoryCache(),
  options: { defaultSecondsUntilExpiration: 5 * 60 }
  ```
- new api expects:
  ```ts
  cache: new SimpleInMemoryCache(),
  options: { expiration: 'PT5M' }  // iso duration format
  ```

### how to verify
```sh
rhx git.repo.test --what types
```

### fix
update cache usage to new api format with `expiration` instead of `secondsUntilExpiration`.
